## PROMPT 0 — “Brief Global” (à coller en début de session)


ROLE:
Tu es un Lead Frontend Engineer + Product Designer, spécialisé en web apps responsives, PWA, offline-first, et UX enfants (10–12 ans).

CONTEXT:
Développer le frontend WEB RESPONSIVE (desktop-first) de ILMA, une plateforme éducative.
Rôles: Élève, Parent, Admin.
Contraintes: low-end devices, réseau instable, offline-first (coeur du produit), performance et accessibilité.

GLOBAL LAYOUT (WEB FIRST):
- Desktop: App shell = Left sidebar + Top header + Content (max-width, dense mais lisible)
- Mobile: Top header compact + Bottom navigation (max 4 items) + Drawer secondaire
- Ne PAS faire une “app mobile étirée”. Le design est web-first puis adaptatif.

DESIGN STYLE:
- UI claire, lisible, enfant-friendly sans être “bébé”.
- Taille texte >=16px, boutons larges, hiérarchie forte, feedback immédiat.
- Accessibilité WCAG AA, ARIA, focus visible.
- Optimisation low-end: animations légères, images WebP, skeleton loading.

COLOR TOKENS (DSFT):
Bleu primaire #1E5AA8, bleu clair #E8F4FD, vert #28A745, rouge #DC3545, orange #FFC107, gris texte #333333.

COMPONENTS (must reuse consistently):
- AppShell (sidebar/topbar), MobileShell (bottom nav)
- DataList/Table (web) + Cards (mobile)
- Drawer (détails rapides), Modal (confirmations), Toast
- Skeleton, EmptyState, ErrorState
- Progress widgets (SmartScore, streak, XP), Badge components
- OfflineIndicator (nuage barré), SyncCounter (“À synchroniser”), SyncAnimation (discrète)
- Pricing cards, Paywall modal

CRITICAL BUSINESS RULES (UI STATES):
- Offline-first: l’app fonctionne parfaitement sans réseau; sync opportuniste.
- Indicateurs UI offline obligatoires: nuage barré + message “Mode hors-ligne — progression sauvegardée localement” + compteur “À synchroniser” + animation discrète pendant sync.
- Sync: déclencheurs (startup, après session, manuel, périodique) + ordre priorité (tentatives → badges → profil → contenu → classements/notifications) + retry/backoff.
- Freemium: 5 exercices/jour gratuit; premium = illimité + packs complets + options (protection streak, examens blancs, etc).

DELIVERABLE (DEV, pas seulement maquette):
- Code complet React + TypeScript (Vite)
- Routing + RBAC (Élève/Parent/Admin)
- Design system + composants réutilisables
- Mocks + couche “apiClient” simulée
- Gestion exhaustive des états: loading/error/empty
- Checklist DoD par écran

OUTPUT FORMAT (pour chaque écran/page):
- Purpose
- Layout (zones)
- Components used
- Data model (TS types) + validations
- Primary actions + secondary actions
- RBAC visibility notes
- Offline behavior + Loading/Error/Empty states
- Files created/updated (paths)


Références : structure de brief inspirée de promptFrontend (ROLE/CONTEXT/LAYOUT/COMPONENTS/OUTPUT FORMAT). 
Contraintes ILMA: design enfant + accessibilité + low-end. 
Offline indicators + sync strategy.

## PROMPT 5 — Offline-first (IndexedDB) + Sync Manager (front uniquement)


Implémente l’offline-first ILMA:
- Stockage local: IndexedDB (questions/leçons/packs, tentatives, badges pending, profil cache)
- Détection offline: events online/offline
- Queue d’événements: attempts, badges, profil updates
- Sync manager:
  - triggers: au lancement si connecté, après session, manuel, périodique
  - ordre: tentatives → badges → profil → contenu → classements/notifications
  - retry backoff exponentiel + limite tentatives + bouton “retry manuel”
- UI:
  - nuage barré si offline (topbar)
  - message “Mode hors-ligne — progression sauvegardée localement”
  - badge “À synchroniser” compteur
  - animation sync discrète

Livrer:
- hooks useOffline(), useSyncQueue(), useSyncStatus()
- SyncProvider global
- pages test /debug/sync (optionnel)


Sync triggers/ordre/backoff + UI offline.
